# 動作の設計

## ファイル形式
- .k
kohanソースファイル

- .kld
１対１でデータを格納し、改行で区分け。
data_A>01
のような記述式

- .kkd
漢字データ格納に特化。読むときは各文字のUnicodeを用いて識別する。

>※設計意図
>漢字とそれ以外の違いを明確にすることで漢字の役割をわかりやすくするため。

- .kファイルを検知 
   1. 変更を検知
      1. コードデータをUnicodeから独自データに変換
         - コンパイラであるなら
            1. 既存言語に書き換えたデータを作成
            2. 依存関係を解決
            3. フォルダ構成のコピー
            4. 適切な形式で保存

# 言語設計
書籍、論、段落、構文、要素・型
で構成されており、左側に位置するものは右側に位置するものすべてを含んでいる。
分解する際に使用する記号については以下に定める。
- 書籍から論
改行
- 論から段落
Setsuzoku(接続詞)
- 段落から構文
Kuten(。) 
- 構文から要素・型
Fuzokugo(助詞・助動詞)
※これは既存のメゾッド名・変数名を除いたものすべてを示す。本質的には前述のものを除くすべてを除いたデータの配列によって付属語として割り振られ、後付けされるものである。

- 課題
ユーザー定義変数の認知方法について
1. 「は」を指定助詞として登録しておくことでその前はユーザー定義であることを示す。
   問題は誤検出が多くなることが目に見えてわかる。
      が、これは変数検出を最後に回せばある程度は解決する。
